# Secrets Rotation Register

Date: 2026-04-07

## Rotation Principles

- Rotate immediately after exposure.
- Prefer short-lived tokens where supported.
- Store customer/integration secrets only in encrypted secret storage.
- Never commit real tokens to README files, seed scripts, manifests, logs, or test fixtures.
- Rotation must include validation and rollback plan.

## Rotation Runbook

1. Create replacement secret.
2. Deploy/update configuration to use replacement.
3. Verify healthy traffic.
4. Revoke old secret.
5. Confirm no old-secret usage in logs.
6. Record rotation date and owner.
